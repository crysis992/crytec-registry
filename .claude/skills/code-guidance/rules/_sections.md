# Sections

This file defines all sections, their ordering, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Component Architecture (components)

**Description:** Server vs Client component decisions, splitting patterns, composition, and organization.

## 2. Server Actions & Mutations (actions)

**Description:** Patterns for server actions with next-safe-action, Zod schemas, error handling, and client consumption.

## 3. Data Fetching (data)

**Description:** Server component fetching, TanStack Query v5 patterns (queries, mutations, infinite scroll, optimistic updates, cache invalidation), Zustand for UI state, and Prisma queries.

## 4. Authentication & Authorization (auth)

**Description:** Session access, role-based access control, route protection, and API key authentication.

## 5. UI & Styling (ui)

**Description:** Tailwind patterns, Shadcn/ui usage, theming system, and responsive design.

## 6. Internationalization (i18n)

**Description:** Translation patterns, formatting, pluralization, and locale management.

## 7. Database (database)

**Description:** Prisma schema conventions, migrations, and query patterns.

## 8. Charts & Visualization (charts)

**Description:** Design guidelines for data visualization and chart components.
